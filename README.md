# Shadow Survivor

Shadow Survivor 是一个使用 AI 协作开发的 2D 生存打怪小游戏。玩家控制角色在夜色竞技场中移动，角色会自动攻击最近的敌人；击败怪物后收集经验晶体，升级时选择技能，目标是在不断增强的怪物压力下存活 90 秒。

这个项目的目标不是把 AI 概念硬塞进游戏玩法，而是做一个通用、可试玩、能展示完整开发过程的小游戏，并在项目文档中说明 AI 如何参与策划、实现、调参、测试和材料整理。

## 玩法

- 使用 `WASD` 或方向键移动。
- 角色会自动攻击最近的敌人。
- 击败敌人会掉落黄色经验晶体。
- 经验满后暂停游戏，并从三个升级选项中选择一个。
- 存活 90 秒即胜利，生命值归零则失败。

## 已实现功能

- 2D 竞技场地图与程序化游戏资产。
- 玩家移动和边界限制。
- 怪物从地图边缘刷出并追踪玩家。
- 自动瞄准和自动射击。
- 子弹、敌人、玩家、经验晶体的碰撞检测。
- 生命值、得分、击杀数、等级、经验条和倒计时 HUD。
- 升级三选一，包括攻速、弹体数量、伤害、移速、生命值和经验吸附范围。
- 胜利 / 失败结算。
- 响应式页面布局，方便本地试玩和在线演示。

## AI 协作过程

这个项目使用 AI 参与了多个阶段，但最终取舍、验证和整合由我完成：

- Claude：用于玩法方向选择、MVP 范围控制和求职叙事判断。
- GPT：用于任务拆解、README 结构、测试清单和表达优化。
- Gemini：适合从玩家体验角度做反向评审，例如节奏、难度和可理解性。
- Codex：用于本地代码实现、组件整合、Phaser 场景开发、构建验证和问题修复。
- Claude Code：适合后续做代码结构检查、状态管理优化和可维护性重构。

我在设计时刻意选择了“通用小游戏”而不是 AI 隐喻游戏。原因是投递对象是游戏公司，作品本身需要先具备正常游戏的完整玩法；AI 协作能力则通过开发流程、文档记录、任务拆解和验证方式体现。
