# Shadow Survivor 投递说明草稿

我开发了一个名为 Shadow Survivor 的 2D 生存打怪小游戏。玩家通过 WASD 或方向键控制角色走位，角色会自动攻击最近的敌人；击败怪物后收集经验晶体，升级时选择技能，最终目标是在不断增强的怪物压力下存活 90 秒。

我选择做通用小游戏，而不是把 AI 概念直接做成玩法，是因为投递公司是游戏方向。我希望作品首先是一个能正常试玩、有明确循环的小游戏，然后再通过开发过程说明我是如何使用 AI 协作完成项目的。

这个项目中，Claude 主要帮助我确定玩法方向和 MVP 范围，GPT 帮我拆解开发任务和整理说明文案，Gemini 可以从玩家体验角度做反向评审，Codex 用于本地代码实现、Phaser 场景开发、构建验证和问题修复，Claude Code 则适合后续进行代码结构检查和重构。

我自己的判断主要体现在三点：第一，控制项目范围，只做移动、刷怪、自动攻击、经验升级和结算，保证短时间内完成闭环；第二，没有直接采用“AI 主题游戏”的方案，而是选择更贴近游戏公司业务的通用玩法；第三，开发完成后通过构建、试玩和功能检查验证项目是否真的可运行，而不是只停留在 AI 生成代码。

项目功能包括：玩家移动、怪物追踪、自动攻击、碰撞检测、经验掉落、升级三选一、HUD 状态展示、胜利/失败结算和响应式页面布局。
